# Organização da documentação

Estrutura canônica atual:

- `docs/architecture/` — arquitetura e desenho da base
- `docs/operations/` — manutenção e operação local
- `docs/validation/` — smoke test, validação manual e registros UX
- `docs/releases/` — baseline oficial e protocolo de release

Arquivos raiz de `docs/` com nomes históricos foram preservados como pontes de compatibilidade.
